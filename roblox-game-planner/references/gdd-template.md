# GDD Template — Required Sections

## 1. Header
- Game title
- Elevator pitch (1-2 sentences — what IS this game?)
- Genre + sub-genre
- Target audience (age range, player type)
- Platform priority (mobile first? PC? Xbox?)

## 2. Core Gameplay Loop
- What does the player do every 30 seconds? (micro loop)
- What does the player do every 5 minutes? (session loop)
- What keeps them coming back tomorrow? (retention loop)
- Diagram: action → reward → upgrade → harder action → bigger reward

## 3. Unique Selling Points
- List 3-5 competing games
- For each: what they do well, what they lack
- How THIS game is different (specific mechanics, not vague)

## 4. Game Pillars (3-4 core principles)
Every design decision must align with these. Example pillars:
- "Constant reward" — player always feels progress
- "Easy to learn, hard to master"
- "Social first" — everything is better with friends
- "Fair monetization" — paying gives convenience, not power

## 5. Player Motivation
- Why keep playing? (progression, collection, competition, social, creativity)
- Retention hooks: daily rewards, streaks, limited events, achievements
- Long-term goals (what takes weeks/months to achieve)

## 6. Session Flow
Step-by-step from join to leave:
1. Player joins → loading screen → spawn at hub
2. Check daily rewards popup
3. Navigate to current zone
4. Core gameplay loop for 10-20 min
5. Return to hub, spend currency on upgrades
6. Check achievements, maybe rebirth
7. Leave (save data)

## 7. First-Time User Experience (FTUE)
Minute-by-minute of the first 5 minutes:
- 0:00 — what they see
- 0:30 — first interaction
- 1:00 — first reward
- 2:00 — tutorial complete
- 3:00 — first real gameplay
- 5:00 — hooked (understand the loop, want more)

## 8. Social Features
- Trading system (what's tradeable, what's not)
- Leaderboards (what stats)
- Group/guild features
- Friend bonuses
- Chat moderation approach

## 9. Art Style
- Visual direction (cartoon, realistic, low-poly, etc.)
- Color palette (warm, cool, vibrant, pastel)
- Reference games/images
- Character style
- Environment style

## 10. Sound Design
- Music genre/mood per area
- SFX style (cartoon pops, realistic, retro)
- UI sounds (click, purchase, error, achievement)

## 11. Technical Notes
- StreamingEnabled? (yes for large maps)
- Max players per server
- Physics intensity (vehicles? projectiles?)
- Data save frequency
- Key performance concerns
